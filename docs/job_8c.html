int job_run ( int64_t  job_id,
const char *  binary,
const char *  upload_dir,
const char *  log_dir 
)

Start a job in a detached background thread.

The thread builds the re-boot command line from the job's database record, opens a log file at log_dir/job_ID.log, then fork/exec's the binary. Subprocess stdout and stderr are piped into the log file. The database is updated to "running" on start and "success"/"failed" on finish.

Parameters
job_idDatabase job ID.
binaryPath to the re-boot executable.
upload_dirDirectory where uploaded hex files are stored.
log_dirDirectory where per-job log files are written.
Returns
0 on success, -1 if the job is already running or thread fails.

◆ job_sse_poll()

void job_sse_poll ( void  )

Poll all active SSE connections and push any new log lines.

Called from the Mongoose timer callback (300 ms interval). Tails each job's log file from the last known offset and sends new lines as SSE "data:" frames. Closes connections whose job has finished.

Must be called from the single-threaded Mongoose event loop.
