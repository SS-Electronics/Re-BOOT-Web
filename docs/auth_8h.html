Detailed Description

Password hashing and session-cookie authentication.

Passwords are stored as "salt:sha256hex" where the salt is 8 random bytes encoded as 16 hex characters. Sessions are random 64-char hex tokens persisted in the SQLite session table with a configurable TTL.

Hash a plaintext password with a fresh random salt.

Reads TOKEN_LEN/4 bytes from /dev/urandom to generate the salt, then computes SHA-256("salt:password") and stores the result as "16-char-salt:64-char-sha256hex" in out.
