Re-BOOT Web

A lightweight C web server for remotely flashing embedded firmware via the Re-BOOT utility. Runs on a Raspberry Pi and provides a Jenkins-style OTA portal. 

╔══════════════════════════════════════╗
║       Re-BOOT Web Server             ║
║                                      ║
║  Language  : C (no runtime deps)     ║
║  HTTP lib  : Mongoose                ║
║  Database  : SQLite3                 ║
║  Auth      : Session + SHA256        ║
║  Streaming : Server-Sent Events      ║
╚══════════════════════════════════════╝

Features

  • Upload Intel HEX firmware files
  • Configure re-boot CLI arguments from UI
  • Live log streaming via Server-Sent Events
  • Stop running flash jobs anytime
  • Job history with timestamps and exit codes
  • Multi-user login (admin / user roles)
  • Single-file SQLite database

Installation

1. Install dependencies

sudo ./install_deps.sh

2. Build

make

3. Run

./reboot-web

Default: http://0.0.0.0:5000

Default login: admin / admin

CLI Options

-l <addr>  Listen address (default 0.0.0.0:5000)
-b <path>  re-boot binary
-d <db>    SQLite database
-u <dir>   uploads directory
-L <dir>   logs directory
-w <dir>   static www directory

REST API

MethodEndpointDescription
POST/api/loginLogin
GET/api/jobsList jobs
POST/api/jobsCreate job
GET/api/jobs/:idJob details
POST/api/jobs/:id/runStart job
POST/api/jobs/:id/stopStop job
GET/api/jobs/:id/streamSSE logs

System Service

sudo systemctl enable reboot-web
